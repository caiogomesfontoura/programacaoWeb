# 📚 Guia de Programação Web

> Um recurso educacional sobre HTML, CSS, JavaScript e desenvolvimento web moderno.

Projeto desenvolvido durante as aulas da disciplina **Programação para Internet** do curso **Sistemas de Informação** da **Faculdade Multivix** (2026).

## 🎯 Objetivo

Consolidar aprendizados e oferecer um material introdutório para alunos estudarem fundamentos de desenvolvimento web.

## 📖 Conteúdo

- **Introdução**: Conceitos fundamentais de programação web
- **HTML**: Estrutura semântica e organização de conteúdo
- **CSS**: Estilo visual e boas práticas de design
- **Layout**: Flexbox e CSS Grid para layouts modernos
- **Recursos**: Links para referências oficiais

## ✨ Recursos Implementados

- ✅ Acessibilidade como função central do projeto
- ✅ Tema escuro/claro com localStorage
- ✅ Navegação suave entre seções
- ✅ Exemplos práticos de código
- ✅ Links para documentação oficial (MDN, W3C, W3Schools)

## ♿ Camada Avançada de Acessibilidade

O projeto agora trata acessibilidade como parte principal da experiência, com recursos que podem ser ajustados em tempo real e salvos no navegador:

- ✅ Link de atalho para pular direto ao conteúdo principal
- ✅ Painel dedicado de preferências de acessibilidade
- ✅ Alto contraste manual
- ✅ Redução de movimento (incluindo suporte a preferências do sistema)
- ✅ Texto maior para melhor legibilidade
- ✅ Foco reforçado para navegação por teclado
- ✅ Destaque de links com sublinhado reforçado
- ✅ Região ARIA live para anunciar mudanças importantes
- ✅ Fechamento por teclado (Esc) e atalho Alt+A para abrir/fechar o painel
- ✅ Compatibilidade com forced colors

## 🚀 Como Usar

1. Clone ou baixe o projeto (ou acesse a página publicada no GitHub Pages)
2. Abra `index.html` em seu navegador
3. Clique no ícone de tema (☀️/🌙) para alternar entre modo claro e escuro
4. Clique em **Acessibilidade** para abrir o painel de preferências

## 🧭 Convenções Didáticas do Código

Para facilitar leitura por alunos de graduação, o projeto segue estas convenções:

- Nomes de variáveis e funções em português claro, descrevendo intenção.
- Identificadores de interface (ids/classes) com termos semânticos e diretos.
- JavaScript curto, com fluxo linear e poucas funções principais.
- CSS com regras consolidadas para reduzir repetição.

## 🧩 Como Ler o Projeto (Ordem Sugerida)

Para alunos de graduação, esta sequência ajuda a entender o projeto com menos fricção:

1. **index.html**: veja a estrutura semântica da página e os elementos de acessibilidade.
2. **style.css**: observe como as classes visuais e os atributos `data-*` controlam aparência e acessibilidade.
3. **script.js**: acompanhe o fluxo de inicialização, tema, menu e painel de acessibilidade.

## 🔎 Núcleo do Projeto

Para manter o projeto simples e funcional, o núcleo foi definido como:

1. Estrutura semântica em HTML.
2. Estilo responsivo com tema claro/escuro.
3. Recursos de acessibilidade acionáveis no painel.
4. JavaScript enxuto para interações principais (menu, tema e acessibilidade).

## ✅ Checklist de Manutenção Didática

Ao alterar o projeto, mantenha estas regras:

1. Use nomes claros em português para variáveis, funções, ids e classes.
2. Prefira lógica direta antes de criar novas camadas de abstração.
3. Evite duplicação excessiva, mas sem perder legibilidade.
4. Atualize este README quando mudar convenções ou estrutura.
5. Preserve recursos de acessibilidade como parte obrigatória da entrega.

## 📌 Observação para Apresentação

Este projeto foi intencionalmente mantido simples para fins didáticos. As decisões de estrutura, nomenclatura e organização priorizam legibilidade e manutenção por alunos iniciantes.
